# Mosquito Flight Trajectory Prediction

월간 데이콘 **모기 비행 궤적 예측 AI 경진대회** 풀이 레포지토리입니다.

> 대회 링크: https://dacon.io/competitions/official/236716/overview/description

---

## 대회 개요

LiDAR 센서로 관측된 모기의 3차원 궤적 데이터를 바탕으로, 시스템 처리 지연(80ms) 이후의 모기 위치를 예측합니다.

- **입력**: -400ms ~ 0ms 구간의 11개 시점 좌표 (40ms 간격)
- **출력**: +80ms 시점의 3차원 좌표 (x, y, z)
- **평가 지표**: R-Hit@1cm

```python
def r_hit(pred, true):
    R_HIT = 0.01
    distance = np.linalg.norm(np.asarray(pred) - np.asarray(true), axis=-1)
    return np.mean(distance <= R_HIT)
```

## 데이터셋

### 좌표계

LiDAR **sensor-local** 3차원 좌표계 기준입니다 (방 기준 절대 좌표 아님).

| 축 | 방향 | 단위 |
|---|---|---|
| x | forward | m |
| y | left | m |
| z | up | m |

### 구성

| 파일/폴더 | 설명 | 샘플 수 |
|---|---|---|
| `data/train/` | 학습용 샘플 CSV | 10,000개 |
| `data/test/` | 평가용 샘플 CSV | 10,000개 |
| `data/train_labels.csv` | 학습 정답 레이블 | 10,000행 |
| `data/sample_submission.csv` | 제출 양식 | - |

### 샘플 CSV 구조 (`train/`, `test/`)

각 파일은 모기 1개체의 관측 시계열입니다.

```
timestep_ms,x,y,z
-400,3.58,-0.08,2.53
-360,3.57,-0.10,2.51
...
0,3.51,-0.19,2.39
```

- 행 수: 11개 (고정)
- `timestep_ms`: -400 ~ 0 (40ms 간격)

### 레이블 CSV 구조 (`train_labels.csv`)

```
id,x,y,z
TRAIN_00001,3.49,-0.21,2.35
...
```

- `id`: 해당 train 샘플 파일명 (확장자 제외)
- x, y, z: 마지막 관측(0ms) 기준 **+80ms** 시점의 실제 좌표

### 환경 다양성

학습/평가 데이터는 **서로 다른 장소 및 환경**을 기반으로 구성됩니다.

- 실내, 복도, 창고, 반실외, 야외 등
- 속도, 가속도, scene ID 등 보조 정보는 **제공되지 않음**
- 오직 과거 좌표 변화만으로 미래 위치를 예측해야 함

## 데이터 분석 (EDA)

train 10,000개 샘플 전수 분석 결과입니다.

### 베이스라인 적중률

| 방법 | R-Hit@1cm |
|---|---|
| 선형 외삽 `x(0) + 2v` | 57.88% |
| 가속도 보정 `x(0) + 2v + 0.6a` | 60.08% |

### 오차 분포

| 방법 | 평균 | 중간값 | p75 | p95 |
|---|---|---|---|---|
| 선형 | 1.294cm | 0.832cm | 1.336cm | 4.480cm |
| 가속도 | 1.298cm | 0.816cm | 1.342cm | 4.484cm |

### 궤적 운동 특성

| 지표 | 평균 | 중간값 | p95 |
|---|---|---|---|
| 속도 (m/40ms) | 0.0256 | 0.0234 | 0.0538 |
| 가속도 크기 (m/40ms²) | 0.00552 | 0.00341 | 0.01825 |
| 궤적 직선성 (1.0=완전 직선) | 0.8945 | 0.9664 | - |

- 중간값 기준 직선성 **0.966** → 대부분이 거의 직선 운동
- 가속도가 속도보다 **10배 이상 작음** → 단기적으로 등속 운동에 가까움

### 케이스 분류

| 케이스 | 수 | 비율 |
|---|---|---|
| 둘 다 HIT | 5,466 | 54.7% |
| 선형만 HIT (가속도 보정이 오히려 악화) | 322 | 3.2% |
| 가속도만 HIT | 542 | 5.4% |
| 둘 다 MISS | 3,670 | 36.7% |

### 주요 결론

- **60%는 단순 물리 공식으로 해결 가능**
- **36.7%는 방향 전환/가속이 큰 어려운 케이스** — 둘 다 MISS인 샘플은 전체 대비 속도(+20%), 가속도(+63%), 직선성(-5%) 차이를 보임
- 선형 vs 가속도 보정의 차이가 **2.2%p** — 물리 공식 튜닝만으로는 한계 존재
- 어려운 케이스의 패턴이 학습 가능한지 여부가 모델 방향 결정의 핵심
